## Executive Summary

The ecosystem is **functionally correct** — 113 tests passing in claw-aie, 509 in repo-transmute, browser review working end-to-end. But there's significant **accidental complexity** that will slow future development if not addressed now.

The biggest wins: delete dead code, deduplicate shared utilities, fix the AIE test runner, and merge two repos that should be one.

## Critical Issues (Do First)

### 1. Dead Code: `src/` in claw-aie (66 files, ~4000 lines)

**Problem:** `claw-aie/src/` is an archived copy of the claw-code TypeScript-to-Python routing layer. Nothing in `aie_integration/` imports from it. It caused a circular import crash (`src/types/__init__.py` shadows Python's built-in `types` module). Tests only pass when `src/` is excluded from `PYTHONPATH`.

**Fix:** Delete `src/` entirely. If the upstream routing layer is needed, import from `claw-aie-harness` (openharness-ai) as a proper dependency.

```bash
# Current: PYTHONPATH=src:. python3 ...  (breaks due to circular import)
# After:   PYTHONPATH=. python3 ...      (clean)

cd claw-aie && rm -rf src/
# Also remove src/ references from SPEC.md and INTEGRATION_SPEC.md
```

**Impact:** -4000 lines, removes circular import, simplifies test setup.

### 2. AIE Tests Can't Run

**Problem:** `agent-interaction-evaluator-repo` has 8 test files with ~2500 lines of tests, but they can't actually run — the import chain pulls in txtai → torch → segfault in this environment. Tests live at `evaluator/tests/` instead of the standard `tests/` location.

**Fix:**
- Move tests from `evaluator/tests/` to `tests/`
- Add `--ignore` markers for heavy ML tests (nli_deberta, txtai_client)
- Create a `tests/test_schema.py` that tests event schema without txtai dependency
- Add CI config that separates "fast tests" (no ML) from "full tests" (with ML)

```toml
# pyproject.toml
[tool.pytest.ini_options]
markers = [
    "heavy: requires torch/txtai",
]
```

### 3. ClawTeam `src/types/` Shadow

**Problem:** The `src/types/__init__.py` file in claw-aie's archived code shadows Python's built-in `types` module, breaking the entire Python runtime when `src/` is on the path.

**Fix:** Resolved by issue #1 (delete `src/`). If `src/` must be kept for reference, rename the conflicting directory.

## High-Impact Simplifications

### 4. Deduplicate: Sanitiser (claw-aie + AIE)

**Problem:** Both repos have a `sanitiser.py` that does the same thing — redact sensitive fields in dicts. The AIE version is more thorough (76 lines vs 9 lines).

**Fix:** claw-aie should import sanitiser from AIE, not maintain its own copy.

```python
# claw-aie/aie_integration/sanitiser.py
from evaluator.sanitiser import sanitise  # delegate to canonical source
```

Or better: extract a shared `claw-aie-utils` package with sanitiser, session, and socket IPC client.

### 5. Deduplicate: Socket IPC Client (5 copies)

**Problem:** The pattern "connect to ailogger.sock, send JSON-RPC, read response" is copy-pasted in:
- `aie_emitter.py` (claw-aie)
- `harness.py` (claw-aie)
- `spawn_hooks.py` (claw-aie)
- `cli.py` (claw-aie)
- `logger_client.py` (AIE)

**Fix:** Single `AIELoggerClient` class, shared across repos.

```python
# In AIE repo: evaluator/logger_client.py already exists
# claw-aie should use it:
from evaluator.logger_client import AIELoggerClient

class AIEEventEmitter(ToolHook):
    def __init__(self, client: AIELoggerClient | None = None):
        self.client = client or AIELoggerClient()
```

### 6. Merge claw-aie + claw-aie-harness

**Problem:** `claw-aie-harness` (openharness-ai, 62K lines) is a full Python port of Claude Code. `claw-aie` (6.5K lines) is the AIE instrumentation layer on top. They're installed as separate packages but claw-aie doesn't import from openharness-ai at all — the `src/` archive in claw-aie is a stale copy of what claw-aie-harness provides.

**Fix:** Make claw-aie properly depend on openharness-ai:
```toml
# claw-aie/pyproject.toml
dependencies = [
    "openharness-ai>=0.1.6",  # proper dep instead of archived copy
    "playwright>=1.50.0",
    ...
]
```
Then delete `src/` and import routing/session primitives from `ohmo.*`.

### 7. Verdict Logic Bug in Browser Review

**Problem:** The reviewer returns "pass" even when HTTP status is 502 or 404. The casa.codeovertcp.com review showed `verdict: "pass"` with `load_status: 502` on all pages.

**Fix:**
```python
# browser_review.py — fix verdict logic
if any(p.error for p in pages):
    verdict = "fail"
elif any(p.load_status >= 400 for p in pages):
    verdict = "fail"  # HTTP errors should fail
elif total_errors > 0 or total_a11y > 5:
    verdict = "warn"
else:
    verdict = "pass"
```

## Medium-Impact Improvements

### 8. Remove opik_client from AIE (291 lines)

**Problem:** `opik_client.py` is a W&B Opik tracking integration. It's only imported conditionally in `logger.py`. Opik is not in pyproject.toml dependencies — it's dead code unless explicitly enabled.

**Fix:** Either add it as an optional dependency or remove it. If keeping, add `opik` to `[project.optional-dependencies]`.

### 9. Consolidate Test Fixtures

**Problem:** claw-aie tests use `asyncio.get_event_loop().run_until_complete()` which is deprecated in Python 3.12+. The `@pytest.mark.asyncio` marker exists but some tests bypass it.

**Fix:** Convert all async tests to use `@pytest.mark.asyncio` consistently, remove manual event loop management.

### 10. Port Sessions Bloat

**Problem:** claw-aie has 36 `.port_sessions/*.json` files committed. These are runtime artifacts, not source code.

**Fix:** Add `.port_sessions/` to `.gitignore` and remove from git tracking.

### 11. Unused Imports

**Problem:** `browser_tools.py` imports `base64` (used), `hashlib` (unused), and several other unnecessary imports. `harness.py` imports `json` twice.

**Fix:** Run `autoflake --remove-all-unused-imports` across the codebase.

## Quality Improvements

### 12. Type Checking

**Problem:** No repo has `mypy` or `pyright` configured. Runtime type errors are caught only by tests.

**Fix:** Add `mypy --strict` to CI for at least claw-aie (smallest, most critical repo).

### 13. Linting

**Problem:** No `ruff` or `flake8` configured. Inconsistent formatting across files.

**Fix:**
```bash
pip install ruff
ruff check --fix aie_integration/
ruff format aie_integration/
```

### 14. Error Handling in Spawn Hooks

**Problem:** `DriftCheckHook` and `OracleEvalHook` silently swallow all exceptions with bare `except Exception: pass`. If the AIE logger is running but returns malformed JSON, the error is invisible.

**Fix:** Log errors to stderr or a structured log, then continue:
```python
except Exception as e:
    import sys
    print(f"[spawn_hook] {self.__class__.__name__} error: {e}", file=sys.stderr)
```

### 15. Browser Tools State Management

**Problem:** `_playwright`, `_browser`, `_page` are module-level globals. If `cleanup_browser()` isn't called between tests, state leaks. The `_ensure_browser()` lazy init pattern works but is fragile.

**Fix:** Use a context manager:
```python
@asynccontextmanager
async def browser_session():
    try:
        page = await _ensure_browser()
        yield page
    finally:
        await cleanup_browser()
```
